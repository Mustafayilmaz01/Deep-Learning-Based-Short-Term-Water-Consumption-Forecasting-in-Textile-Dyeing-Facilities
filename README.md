## Overview

This repository benchmarks **9 forecasting models** ‚Äî ranging from classical statistical methods (ARIMA, SARIMA) to deep learning architectures (LSTM-FCN, InceptionTime) and kernel methods (ROCKET) ‚Äî on a monthly industrial wastewater (W-Water) dataset spanning **January 2011 to December 2022 (144 months)**.

**Key design decisions:**
- Strict chronological train/validation/test split with **no data leakage**: scalers and correlation statistics are computed exclusively on training data.
- Sliding-window feature engineering (W = 12 months) with 3 exogenous input features selected via correlation analysis.
- 95% bootstrap confidence intervals (n = 1,000) for all model predictions.
- Multi-seed stability evaluation (5 seeds) for stochastic deep learning models.
- Residual autocorrelation tests (Ljung-Box) for all models.

**Data split (raw months ‚Üí sliding-window samples):**

| Split | Period | Raw months | Samples (W=12) |
|-------|--------|-----------|----------------|
| Train | Jan 2011 ‚Äì Dec 2019 | 108 | 96 |
| Validation | Jan 2020 ‚Äì Dec 2021 | 24 | 24 |
| Test | Jan 2022 ‚Äì Dec 2022 | 12 | 12 |

## Installation

```bash
# 1. Clone the repository
git clone https://github.com/YOUR_USERNAME/Mustafa_Water_Forecasting.git
cd Mustafa_Water_Forecasting

# 2. Create and activate a virtual environment (recommended)
python -m venv venv
source venv/bin/activate          # Linux / macOS
venv\Scripts\activate             # Windows

# 3. Install all dependencies
pip install -r requirements.txt
```

**`requirements.txt`:**

```
numpy>=1.23
pandas>=1.5
matplotlib>=3.6
scikit-learn>=1.2
statsmodels>=0.14
xgboost>=1.7
sktime[all_extras]>=0.24
tensorflow>=2.12,<2.16
seaborn>=0.12
```

> **Note on `sktime[all_extras]`:** This installs ROCKET, LSTM-FCN, and InceptionTime. If you only need the statistical baselines and skip deep learning models, `pip install sktime` suffices and you can omit TensorFlow.

### Common installation issues

- **TensorFlow on Apple Silicon (M1/M2/M3):** Use `tensorflow-macos` and `tensorflow-metal` instead of the standard `tensorflow` package.
- **Windows users:** If `sktime[all_extras]` fails, try installing `sktime` first, then `tensorflow` separately.
- **`pm.auto_arima` not found:** Ensure `pmdarima` is installed ‚Äî it is pulled in by `sktime[all_extras]`, but can also be installed manually with `pip install pmdarima`.

## How to Run

The pipeline consists of two sequential steps.

### Step 1 ‚Äî Prepare sliding-window data

```bash
python prepare_lstm_data.py
```

This script reads `data/fabrika_clean.csv`, applies the 3-way chronological split, computes the train-only correlation heatmap, and generates all arrays in `data_lstm/`. It must be run **before** Step 2.

Expected output:
```
Veri okundu
  Zaman araligi : 2011-01 -> 2022-12
  Toplam ay     : 144
...
OZET:
  Train : 2011-01 -> 2020-12  (120 ay | pencere sonrasi 108 ornek)
  Val   : 2021-01 -> 2021-12  (12 ay  | pencere sonrasi 12 ornek)
  Test  : 2022-01 -> 2022-12  (12 ay  | pencere sonrasi 12 ornek)
  X     : (132, 12, 3)
  y     : (132,)
```

### Step 2 ‚Äî Train models and evaluate

```bash
python run_sktime_from_prepared.py
```

This script loads `data_lstm/`, runs hyperparameter tuning, trains all 9 models, computes metrics with bootstrap confidence intervals, runs residual tests, and saves all results and figures to `results_sktime_wwater_ieee/`.

**Expected total runtime:** ~5‚Äì10 minutes on a modern CPU (dominated by InceptionTime at ~97 s and LSTM-FCN at ~69 s).

### Optional ‚Äî Regenerate correlation heatmaps only

```bash
python correlation.py
```

Generates two heatmap PNGs at 600 DPI: one for all features and one for the 3 selected features vs. the target.

## Reproducibility Notes

- **Random seed:** All stochastic components use seed `42` by default (NumPy, Python `random`, TensorFlow). The seed is set at the top of `run_sktime_from_prepared.py` and propagated to all model constructors.
- **No data leakage:** The `MinMaxScaler` for input features and the target scaler are fitted exclusively on the training split and applied to validation/test splits via `transform` only.
- **Multi-seed runs:** LSTM-FCN and InceptionTime are additionally evaluated with seeds `[42, 123, 7, 99, 2024]` to quantify variance due to random weight initialization.
- **Operating system:** Results were obtained on Windows 10/11 x64. Minor numerical differences (< 0.01%) may occur across platforms due to floating-point non-determinism in TensorFlow.
- **TensorFlow determinism flags** are set at startup (`TF_DETERMINISTIC_OPS=1`, `TF_CUDNN_DETERMINISTIC=1`) to maximize reproducibility on GPU-enabled machines.
